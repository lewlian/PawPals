<task type="auto">
  <name>Task 2: Create type-safe environment validation</name>
  <files>

  </files>
  <action>
    1. Create src/config/env.ts with zod schema validation:
       ```typescript
       import { z } from 'zod';

       const envSchema = z.object({
         // Bot configuration
         BOT_TOKEN: z.string().min(1, 'BOT_TOKEN is required'),

         // PostgreSQL configuration
         DB_HOST: z.string().default('localhost'),
         DB_PORT: z.string().regex(/^\d+$/).transform(Number).default('5432'),
         DB_NAME: z.string().default('pawpals'),
         DB_USER: z.string().default('postgres'),
         DB_PASSWORD: z.string().default('postgres'),

         // Redis configuration
         REDIS_URL: z.string().default('redis://localhost:6379'),

         // Environment
         NODE_ENV: z.enum(['development', 'test', 'production']).default('development'),
       });

       export type Env = z.infer<typeof envSchema>;

       export function validateEnv(): Env {
         const result = envSchema.safeParse(process.env);

         if (!result.success) {
           console.error('Environment validation failed:');
           console.error(result.error.format());
           process.exit(1);
         }

         return result.data;
       }
       ```

    2. Create .env.example with placeholder values (NOT real secrets):
       ```

       DB_USER=postgres
       DB_PASSWORD=postgres

       # Redis Configuration
       REDIS_URL=redis://localhost:6379

       # Environment
       NODE_ENV=development
       ```

    IMPORTANT:
    - validateEnv() must call process.exit(1) on validation failure for fail-fast behavior
    - All secrets have sensible development defaults EXCEPT BOT_TOKEN (no default, required)
    - Export both the Env type and validateEnv function for use in other modules
