<task type="auto">
  <name>Task 1: Create the profile creation wizard scene</name>
  <files>
    src/bot/scenes/createDogProfile.ts
  </files>
  <action>
    Create `src/bot/scenes/createDogProfile.ts` implementing a WizardScene with these steps:

    **Step 0 - Entry:**
    - Initialize wizard state: `ctx.wizard.state.dogData = {}`
    - Send: "Let's create a profile for your dog!\n\nWhat is your dog's name?"
    - Call `ctx.wizard.next()`

    **Step 1 - Receive Name:**
    - Use Composer for text handling
    - Validate: 1-50 characters, trim whitespace
    - Store in `ctx.wizard.state.dogData.name`
    - Show size selection with inline keyboard:
      - "Small (under 10kg)" -> callback_data: "size_Small"
      - "Medium (10-25kg)" -> callback_data: "size_Medium"
      - "Large (over 25kg)" -> callback_data: "size_Large"
    - Reply: "Great! {name} is a lovely name.\n\nWhat size is {name}?"
    - Call `ctx.wizard.next()`

    **Step 2 - Breed Selection:**
    - This step waits for size callback or breed text search
    - Action handler `size_(.+)` stores size, shows breed prompt:
      "Now, type your dog's breed to search, or browse popular breeds:"
      With button: "Browse popular breeds" -> `breed_browse_0`
    - Action handler `breed_browse_(\d+)` shows paginated breed list (6 per page)
    - Text handler during this step: calls searchBreeds(), shows matching buttons
    - Action handler `breed_select_(.+)` stores breed and moves to age step

    **Step 3 - Age Input:**
    - Text handler validates integer 0-30
    - Store in `ctx.wizard.state.dogData.age`
    - Move to confirmation step

    **Step 4 - Confirmation:**
    - Display all collected data:
      "Please confirm your dog's profile:\n\nName: {name}\nSize: {size}\nBreed: {breed}\nAge: {age} years\n\nIs this correct?"
    - Buttons: "Yes, save profile" -> `confirm_save`, "No, start over" -> `confirm_restart`

    **Action handlers:**
    - `confirm_save`:
      - Get telegram ID from ctx.from.id
      - Call findOrCreateUser(telegramId, firstName, username)
      - Call createDog(user.id, dogData)
      - Reply: "{name}'s profile has been created!\n\nUse /profile to view and manage your dog profiles."
      - ctx.scene.leave()
    - `confirm_restart`: ctx.scene.reenter()

    **Global handlers:**
    - Command /cancel at any step: "Profile creation cancelled." + ctx.scene.leave()

    **Implementation notes:**
    - Always call ctx.answerCbQuery() immediately in action handlers (prevents loading spinner)
    - Use ctx.editMessageText where appropriate (replaces button message)
    - Import from: telegraf (Scenes, Markup, Composer), data/breeds.js, repositories
    - Use BREEDS_PER_PAGE = 6 constant
    - Handle edge case: user sends text when keyboard expected (re-prompt)
  </action>

  </verify>

    - Breed search with text filter + browse pagination

    - Confirmation with save/restart options

  </done>
