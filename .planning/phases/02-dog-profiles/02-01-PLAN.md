  <action>
    1. Install required packages: `npm install @telegraf/session kysely`

    2. Create migration file `src/db/migrations/0002-users-dogs.sql` with:
       - users table:
         - id SERIAL PRIMARY KEY
         - telegram_id BIGINT NOT NULL UNIQUE (Telegram IDs exceed INT max)
         - first_name VARCHAR(255)
         - username VARCHAR(255)
         - created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
         - updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
       - dogs table:
         - id SERIAL PRIMARY KEY
         - user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE
         - name VARCHAR(100) NOT NULL
         - size VARCHAR(10) NOT NULL CHECK (size IN ('Small', 'Medium', 'Large'))
         - breed VARCHAR(100) NOT NULL
         - age INTEGER CHECK (age >= 0 AND age <= 30)
         - created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
         - updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
       - Indexes: idx_dogs_user_id ON dogs(user_id), idx_users_telegram_id ON users(telegram_id)

    3. Run migration: `npm run db:migrate`

    Use IF NOT EXISTS for idempotency (matches existing migration pattern).
  </action>
  <verify>
    Run: `psql -d pawpals -c "\dt"` shows users and dogs tables
    Run: `psql -d pawpals -c "\d users"` shows BIGINT for telegram_id
    Run: `psql -d pawpals -c "\d dogs"` shows CHECK constraint on size
  </verify>
  <done>
    - users and dogs tables exist in database
    - Foreign key from dogs.user_id to users.id
    - CHECK constraint on dogs.size limits to Small/Medium/Large
    - Indexes created for efficient lookups
  </done>

       - Import pool from '../client.js'
       - findUserByTelegramId(telegramId: number): Promise<User | null>
         - SELECT * FROM users WHERE telegram_id = $1
         - Return null if not found
       - findOrCreateUser(telegramId: number, firstName?: string, username?: string): Promise<User>
         - INSERT ... ON CONFLICT (telegram_id) DO UPDATE SET first_name = EXCLUDED.first_name, username = EXCLUDED.username, updated_at = CURRENT_TIMESTAMP
         - RETURNING * to get the user

       Define User interface: { id: number; telegramId: number; firstName: string | null; username: string | null; createdAt: Date; updatedAt: Date }

    3. Create `src/db/repositories/dogRepository.ts`:
       - Import pool from '../client.js'
       - Import Dog, CreateDogInput, UpdateDogInput from '../../types/dog.js'
       - createDog(userId: number, input: CreateDogInput): Promise<Dog>
         - INSERT INTO dogs (user_id, name, size, breed, age) VALUES ($1, $2, $3, $4, $5) RETURNING *
       - findDogsByUserId(userId: number): Promise<Dog[]>
         - SELECT * FROM dogs WHERE user_id = $1 ORDER BY created_at ASC
       - findDogById(id: number): Promise<Dog | null>
         - SELECT * FROM dogs WHERE id = $1
       - updateDog(id: number, input: UpdateDogInput): Promise<Dog | null>
         - Build dynamic SET clause for provided fields only
         - UPDATE dogs SET ... WHERE id = $1 RETURNING *
       - deleteDog(id: number): Promise<boolean>
         - DELETE FROM dogs WHERE id = $1
         - Return rowCount > 0

       Map database snake_case to camelCase in return objects (user_id -> userId, created_at -> createdAt)
