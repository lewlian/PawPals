<task type="checkpoint:human-action" gate="blocking">
  <action>Set up Railway project and Supabase database</action>
  <instructions>
**Step 1: Create Railway Project**

1. Go to https://railway.app and sign in (create account if needed)
2. Click "New Project"
3. Select "Deploy from GitHub repo"
4. Authorize Railway to access your GitHub account
5. Select the PawPals SG repository
6. Railway will start initial deployment (it may fail - that's expected)

**Step 2: Get Railway Domain**

1. In Railway project dashboard, go to Settings -> Networking
2. Click "Generate Domain" under Public Networking
3. Copy the generated domain (e.g., `pawpals-sg-production.up.railway.app`)
4. Save this - you'll need it for WEBHOOK_DOMAIN

**Step 3: Set Up Supabase (if not already)**

1. Go to https://supabase.com and sign in
2. Create new project or use existing
3. Go to Project Settings -> Database
4. Copy the Connection String (URI format) - looks like:
   `postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres`
5. Run migrations in Supabase SQL Editor if needed (check existing tables)

**Step 4: Generate Webhook Secret**

Run this command locally to generate a secure secret:
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```
Copy the output (64-character string).

**Step 5: Get Your Telegram Chat ID**

1. Message @userinfobot on Telegram with /start
2. It will reply with your user ID (a number like 123456789)
3. Or: temporarily add `console.log(ctx.from.id)` to your bot's start handler, message your bot, check Railway logs

**Step 6: Configure Railway Environment Variables**

In Railway project dashboard -> Variables tab, add:

| Variable | Value |
|----------|-------|
| BOT_TOKEN | Your bot token from .env |
| DATABASE_URL | Supabase connection string from Step 3 |
| NODE_ENV | production |
| WEBHOOK_DOMAIN | Railway domain from Step 2 (without https://) |
| WEBHOOK_SECRET | Generated secret from Step 4 |
| ADMIN_CHAT_ID | Your Telegram ID from Step 5 |

Note: DATABASE_URL replaces the individual DB_HOST, DB_PORT, etc. vars.
If using DATABASE_URL, update env.ts to accept it (or keep using individual vars and parse Supabase URL accordingly).

**Step 7: Trigger Redeploy**

1. After adding all variables, Railway may auto-redeploy
2. If not, click "Redeploy" in the Deployments tab
3. Watch deployment logs for "Bot running in webhook mode"

  <resume-signal>Type "deployed" once Railway shows successful deployment and you see "Bot running in webhook mode" in logs</resume-signal>
</task>
