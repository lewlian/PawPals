  <action>
Modify the existing bot.catch() handler in src/bot/index.ts to send error notifications to admin.

The existing handler (lines 34-39) does:
1. console.error the error
2. Try to reply to user

Enhance it to ALSO send admin notification in production:

1. Keep existing behavior (console.error + ctx.reply)

2. Add admin notification block AFTER the user reply:
   ```typescript
   // Send error details to admin (production only)
   if (env.NODE_ENV === 'production' && env.ADMIN_CHAT_ID) {
     try {
       const errorMessage =
         'Bot Error\n\n' +
         `Type: ${ctx.updateType}\n` +
         `Error: ${err instanceof Error ? err.message : String(err)}\n` +
         `User: ${ctx.from?.id || 'unknown'}\n` +
         `Time: ${new Date().toISOString()}\n\n` +
         `Stack: ${err instanceof Error ? err.stack?.substring(0, 500) : 'N/A'}`;

       await bot.telegram.sendMessage(env.ADMIN_CHAT_ID, errorMessage);
     } catch (notifyErr) {
       // CRITICAL: Never throw from error handler - would cause infinite loop
       console.error('Failed to send admin notification:', notifyErr);
     }
   }
   ```

Important:
- The handler needs to be async (change callback to async)
- Access env.NODE_ENV and env.ADMIN_CHAT_ID (env is already in scope from validateEnv())
- Handle both Error objects and unknown thrown values (err instanceof Error check)
- Truncate stack to 500 chars to avoid Telegram message length limits
- Wrap notification in try-catch and ONLY log failures - never throw
- Do NOT use emojis in error messages (user preference)
  </action>
