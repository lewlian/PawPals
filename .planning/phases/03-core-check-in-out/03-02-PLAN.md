  <action>
Create `src/bot/scenes/checkInWizard.ts` following the createDogProfile.ts wizard pattern.

The wizard has 5 steps:

**Step 0 (stepEntry)**: Entry point
- Initialize wizard state: `ctx.wizard.state = {}`
- Reply with location request message and keyboard:
  ```typescript
  Markup.keyboard([
    [Markup.button.locationRequest('Share Location')],
    [Markup.button.text('Cancel')]
  ]).oneTime().resize()
  ```
- Advance to next step

**Step 1 (stepLocation)**: Handle location message
- On 'location' event:
  - Get latitude/longitude from `ctx.message.location`
  - Reply "Checking nearby dog runs..." with `Markup.removeKeyboard()`
  - Call `validateGeofence(lat, lon)`
  - If NOT within geofence:
    - Calculate distanceKm = (distance / 1000).toFixed(1)
    - Reply error: "You're too far from any dog run! Nearest: {name} ({distanceKm}km away). You must be within 200 meters to check in."
    - Leave scene
  - If within geofence:
    - Store locationId and locationName in wizard state
    - Reply "Location confirmed: {locationName}"
    - Advance to next step
- On 'text' event:
  - If text === 'Cancel': Reply "Check-in cancelled.", leave scene
  - Otherwise: Reply "Please share your location using the button, or tap Cancel."

**Step 2 (stepDogs)**: Fetch and display dog selection
- On 'message' event (auto-triggered):
  - Get telegramId from ctx.from.id
  - Find or create user
  - Get dogs via `findDogsByUserId(user.id)`
  - If no dogs: Reply "You need to create a dog profile first! Use /profile to create one." and leave scene
  - Store userId and allDogIds in wizard state
  - Build inline keyboard with dogs: `dog_{id}` callback for each dog
  - If multiple dogs, add "All Dogs" button at top with callback `dog_all`
  - Reply "Which dog(s) are you checking in?" with keyboard
  - Advance to next step

**Step 3 (stepDogCallback)**: Handle dog selection
- On action `/^dog_(\d+|all)$/`:
  - answerCbQuery
  - If selection === 'all': set selectedDogIds = allDogIds from state
  - Otherwise: set selectedDogIds = [parsed dogId]
  - Build duration keyboard:
    - '15 minutes' -> 'dur_15'
    - '30 minutes' (mark with star as default) -> 'dur_30'
    - '60 minutes' -> 'dur_60'
  - editMessageText "How long will you stay?" with duration keyboard
  - Advance to next step

**Step 4 (stepDuration)**: Handle duration and create session
- On action `/^dur_(\d+)$/`:
  - answerCbQuery
  - Parse durationMinutes from match
  - Get state (userId, locationId, selectedDogIds, locationName)
  - Call `createSession(userId, locationId, durationMinutes)`
  - Call `addDogsToSession(session.id, selectedDogIds)`
  - Calculate expiryTime for display (toLocaleTimeString 'en-SG' format)
  - Get dog names for confirmation message
  - editMessageText confirmation:
    ```
    Check-in successful!

    Location: {locationName}
    Dog(s): {dogNames}
    Duration: {durationMinutes} minutes
    Auto check-out at: {expiryTime}

    Use /checkout to end your session early.
    ```
  - Leave scene

**Global handlers:**
- command('cancel'): Reply "Check-in cancelled." with Markup.removeKeyboard(), leave scene

Export as `checkInWizard`.
